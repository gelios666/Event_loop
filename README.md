# Star Wars ETL (SWAPI → SQLite)

## 📌 Описание проекта

Проект реализует асинхронную загрузку данных о персонажах вселенной Star Wars из API SWAPI с последующим сохранением в базу данных SQLite.

Реализован полный ETL-процесс:

* **Extract** — получение данных из API
* **Transform** — нормализация и преобразование данных
* **Load** — загрузка в базу данных

📌 README.md соответствует текущей реализации проекта и является источником описания архитектуры.

## 🧱 Схема базы данных

Используется база данных **SQLite** (через библиотеку `aiosqlite`).

Таблица `people`:

| Поле       | Тип     | Описание         |
| ---------- | ------- | ---------------- |
| id         | INTEGER | PRIMARY KEY      |
| name       | TEXT    | Имя персонажа    |
| birth_year | TEXT    | Год рождения     |
| eye_color  | TEXT    | Цвет глаз        |
| gender     | TEXT    | Пол              |
| hair_color | TEXT    | Цвет волос       |
| homeworld  | TEXT    | Название планеты |
| mass       | REAL    | Масса            |
| skin_color | TEXT    | Цвет кожи        |

### 🔑 Ключ записи

Поле `id` является **единственным идентификатором записи**:

* объявлено как `PRIMARY KEY`
* используется для определения уникальности записи

Другие поля (например `name`) **не имеют ограничения уникальности** и не участвуют в определении конфликта.

## 🔄 Логика вставки (Upsert)

В загрузчике используется конструкция:

```sql
INSERT INTO people (...)
ON CONFLICT(id) DO UPDATE
```

Это означает:

* если запись с таким `id` уже существует → она **обновляется**
* если записи нет → она **создаётся**
* дубликаты **не возникают**

📌 Поведение полностью согласовано со схемой таблицы (`PRIMARY KEY(id)`).

## 🔄 Логика работы приложения

### 1. Получение списка персонажей (Extract)

* используется пагинация API (`next`)
* извлекаются все `id` персонажей

### 2. Загрузка данных

* выполняются асинхронные HTTP-запросы (`aiohttp`)
* используется `asyncio.gather`
* ограничение параллельности через `Semaphore`

### 3. Нормализация данных (Transform)

* `mass` приводится к типу `float` (или `NULL`)
* `homeworld`:

  * из URL преобразуется в название планеты
  * через дополнительный API-запрос

### 4. Сохранение в базу данных (Load)

* данные записываются в таблицу `people`
* используется `ON CONFLICT(id)` для обновления существующих записей
