# Star Wars ETL (SWAPI → SQLite)

## Описание проекта

В данном проекте реализована асинхронная загрузка данных о персонажах вселенной Star Wars из API SWAPI с последующим сохранением в базу данных SQLite.

Проект демонстрирует:

* работу с внешним API
* асинхронное программирование (`asyncio`, `aiohttp`)
* работу с базой данных (`aiosqlite`)
* базовый ETL-пайплайн (Extract → Transform → Load)

---

## Стек технологий

* Python 3.10+
* asyncio
* aiohttp
* aiosqlite
* SQLite

## Схема базы данных

Таблица `people`:

| Поле       | Тип     | Описание       |
| ---------- | ------- | -------------- |
| id         | INTEGER | ID персонажа   |
| name       | TEXT    | Имя            |
| birth_year | TEXT    | Год рождения   |
| eye_color  | TEXT    | Цвет глаз      |
| gender     | TEXT    | Пол            |
| hair_color | TEXT    | Цвет волос     |
| homeworld  | TEXT    | Родная планета |
| mass       | REAL    | Масса          |
| skin_color | TEXT    | Цвет кожи      |

* `id` — PRIMARY KEY
* Используется `ON CONFLICT(id)` для предотвращения дубликатов

---

## Логика работы (ETL)

### 1. Extract (извлечение данных)

* Получение всех персонажей через пагинацию API
* Асинхронные HTTP-запросы

### 2. Transform (обработка)

* Преобразование данных к нужному формату
* Конвертация `mass` → `float`
* Получение названия планеты (`homeworld`) по URL

### 3. Load (загрузка)

* Сохранение в SQLite
* Обновление записей при повторной загрузке (`ON CONFLICT`)

## Запуск проекта

### 1. Установить зависимости

```bash
pip install aiohttp aiosqlite
```

### 2. Применить миграцию

```bash
python migration.py
```

### 3. Загрузить данные

```bash
python loader.py
```

---

## Повторный запуск

* Миграцию можно запускать повторно (используется `IF NOT EXISTS`)
* Данные не дублируются благодаря `ON CONFLICT(id)`
